JavaScript Variables & Data Types

Declaring Variables: var, let, const

var declares variables with function or global scope and allows redeclaration. let declares block-scoped variables and doesn't allow redeclaration in the same scope. const declares block-scoped constants that cannot be reassigned.

var a = 10;       // function/global scoped, redeclarable
let b = 20;       // block scoped, no redeclaration
const c = 30;     // block scoped, constant value
var a = 10; // function/global scoped
let b = 20; // block scoped
const c = 30; // block scoped, constant

JavaScript Data Types

JavaScript has 7 primitive data types and object type:

  • Primitive types: null, undefined, boolean, number, string, symbol, bigint
  • Object type: objects, arrays, functions, etc.
let n = null;                  // null
let u;                         // undefined (declared but no value)
let isTrue = true;             // boolean
let num = 123;                 // number
let str = "hello";             // string
let sym = Symbol("id");        // symbol
let bigIntNum = 9007199254740991n; // BigInt
let obj = { name: "John" };    // object
n = null
u = undefined
isTrue = true
num = 123
str = "hello"
sym = Symbol("id")
bigIntNum = 9007199254740991n
obj = { name: "John" }

Primitive vs Reference Values

Primitive values are stored directly on the stack and include all primitives listed above. Reference values are objects, stored on the heap, and the variable holds a reference (pointer) to the heap.

Example:

let x = 42;           // primitive, value stored in stack
let y = "Hello";      // primitive, value stored in stack
let obj = { name: "John", age: 30 };  // object, reference stored in stack, actual object in heap

let obj2 = obj;       // obj2 points to same object in heap
obj2.age = 31;        // modifies object on heap, visible via obj too
